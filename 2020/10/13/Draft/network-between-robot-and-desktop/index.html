在ROS下使用zeroconf配置多机通信网络

本文最后更新于：2020年10月13日 晚上

1. 问题起源

在分别使用路由器和手机热点做测试时，ip地址的不同导致需要频繁修改.bashrc文件

2. 解决方案

系统要求：ubuntu

原理：Zeroconf

2.1 hostname

首先使用以下命令分别获取电脑和移动机器人的 hostname

# on the desktop
$ hostname
my_desktop

#on the robot
$ hostname
robot

2.2 Zeroconf hostname

电脑和移动机器人的 Zeroconf hostname 分别为

# desktop
my_desktop.local

#robot
robot.local

2.3 测试连通性

# on the desktop
$ ping robot.local

2.4 配置 .bashrc

# on the desktop
$ vim ~/.bashrc
export ROS_HOSTNAME=my_desktop.local

#on the robot
$ vim ~/.bashrc	
export ROS_HOSTNAME=robot.local
export ROS_MASTER_URI=http://my_desktop.local:11311
